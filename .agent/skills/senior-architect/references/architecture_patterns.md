# Architecture Patterns

## 1. Microservices vs Monolith
Guidelines for choosing between a monolithic architecture and microservices based on team size and complexity.

## 2. Event-Driven Architecture
Using message brokers (Kafka, RabbitMQ) for decoupled system communication.

## 3. Layered Architecture
Standard 3-tier and N-tier patterns for enterprise applications.

## 4. Clean Architecture
Implementing dependency inversion and entities/use-cases/adapters boundaries.
